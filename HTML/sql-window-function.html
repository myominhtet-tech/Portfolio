Introduction to Window Functions

Window functions တွေက powerful feature တွေဖြစ်ပြီးတော့ SQL တွေရဲ့အချို့ variations တွေမှာအလုပ်လုပ်နိုင်တာပဲဖြစ်ပါတယ်။window functions တွေက entire-result set တစ်ခုရဲ့ context တစ်ခုထဲမှာရှိတဲ့ row ကို analyze ပြုလုပ်ပေးတာပဲဖြစ်ပါတယ်။အခြားစကားလုံးနဲ့ပြောရရင်တော့ query ထဲမှာရှိတဲ့ row တစ်ခုနဲ့ အခြား row တစ်ခုကိုကျွန်တော်တို့နှိုင်းယှဉ်နိုင်ပါတယ်။ူဥပမာကျွန်တော်တို့က row တစ်ခုဆီရဲ့ total တွေရဲ့ percentage တွေကိုတွက်ထုပ်မယ် သို့မဟုတ် report တွေတဲ့ကို average တွေကိုရွှေ့ပြီးထပ်ထည့်ပေးတာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့မှာ company ရဲ့monthly sales တွေကို region အရ report တွေကို listing ပြုလုပ်ရမှာရှိပြီးတော့ region တစ်ခုရဲလတစ်လမှာရှိတဲ့ result တွေရဲ့ row တစ်ခုပဲဖြစ်ပါတယ်။ကျွန်တော်တို့က region sales တွေတစ်ခုဆီကိုတွက်ချက်ဖို့အတွက် window function တွေကိုအသုံးပြုနိုင်ပါတယ်။နောက်ပြီးလအလိုက်ရောင်းရတာတွေရဲ့ percentage တွေအဖြစ်လည်းပဲတွက်ထုပ်နိုင်ပါတယ်။ကျွန်ောတ်တို့က window functions တွေကို function call တွေအသုံးပြုပြီးတော့ query တစ်ခုရဲ့အတွင်းမှာပဲ multiple level ပေါ်မှာ aggregate ပြုလုပ်ဖို့အတွက်အသုံးပြုနိုင်ပါတယ်။SQL Server, Oracle , PostgreSQL တွေနဲ့နောက်ထပ် similar database systems တွေမှာလည်းပဲ window functions တွေကိုထောက်ပံ့ပေးနိုင်ပါတယ်။ကျွန်တော်တို့က SQL တွေရဲ့ variations တွေအမျိုးမျိုးနဲ့အလုပ်လုပ်တဲ့အခါမှာ window functions တွေကကျွန်တော်တို့ရဲ့ SQL skills ကိုနောက်ထပ် Level တစ်ခုကိုရောက်ဖို့အတွက်ကောင်းမွန်တဲ့နည်းလမ်းတွေပဲဖြစ်ပါတယ်။ PostgreSQL ကတော့ open source database တစ်ခုပဲဖြစ်ပါတယ်။အဲ့တာကြောင့်ကျွန်တော်တို့တွေက free download ပြုလုပ်နိုင်ပါတယ်။ ကျွန်တော်တို့က database ထဲမှာရှိတဲံအဖြေတွေကိုဖော်ပြဖို့အတွက်တွေနဲ့ တွက်ထုပ်ဖို့အတွက်တွေကိုနောက်ထပ်additional report တွေရေးနေစရာမလိုပဲ window function တွေကိုအသုံးပြုပြီးတော့ဖော်ပြနိုင်ပါတယ်။

Moving Beyond the Row_ The OVER Keyword

ကျွန်တော်တို့တစ်တွေက Query တွေထဲမှာ sum, count တွေနဲ့ average ဆိုတဲ့ aggregate functions တွေကိုအသုံးပြုတဲ့အခါမှာအဲ့တာတွေက record တွေကိုအသုံးပြုပြီးတော့ result တွေကိုတွက်ထုပ်ပါတယ်။အဲ့ဒီ record တွေက GROUP BY statement မှာရှိတဲ့သူတို့ရဲ့ row တွေနဲ့ဆက်နွယ်နေတာပဲဖြစ်ပါတယ်။သူတို့တွေရဲ့ grouping ရဲ့အပြင်မှာရှိတဲ့ records တွေရဲ့မည်သည့် value ကိုမှလက်ခံမှာမဟုတ်ပါဘူး။ အဲ့တာကြောင့် return ပြန်လာတဲ့ values တွေကို overall results ရဲ့ context ထဲမှာနေရာချထားမှာပဲဖြစ်ပါတယ်။over keyword နဲ့ကျွန်တော်တို့က window ကိုဖွင့်နိုင်မှာဖြစ်ပြီးတော့အဲ့ဒီ aggregate functions တွေကမြင်နိုင်ပြီးတော့ query တွေရဲ့ result တွေအားလုံးကို access ပြုလုပ်နိုင်မှာပဲဖြစ်ပါတယ်။အဲ့ဒီပြောင်းလဲမှုတွေကပုံမှန် aggregate function တွေကို window function အဖြစ်ပြောင်းပေးတာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့တွေ over keyword ကိုအသုံးပြုတဲ့အခါမှာ information တွေကိုထပ်ပြီးတော့ထည့်နိုင်ပါတယ်။average့ိုလိုချင်ရင်တော့ average function ကိုထည့်ပြီးတော့အသုံးပြုပေးရပါတယ်။နောက်ထပ်ကျွန်တေ်ာတို့က row တွေအကုန်လုံးကို function ကလက်ခံနိုင်ဖို့အတွက် over keyword ကိုအသုံးပြုနိုင်ပါတယ်။over keyword ရဲ့နောက်မှာတော့ empty parentheses ကိုထည့်ပေးရမှာပဲဖြစ်ပါတယ်။parentheses က space ကိုတည်ဆောက်ပေးတာပဲဖြစ်ပါတယ်။နောက်ပြီးကျွန်တော်တို့က sum of sum of checkmate ဆိုတဲ့ function ကိုလည်း empty parentheses နဲ့တွဲပြီးတော့အသုံးပြုနိုင်ပါတယ်။
