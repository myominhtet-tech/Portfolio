JavaScript Outside of the Browser

JavaScript ကကျွန်တော်တို့ world ပေါ်မှာ popular အဖြစ်ဆုံးနဲ့ versatile အဖြစ်ဆုံး programming language တစ်ခုပဲဖြစ်ပါတယ်။ဒီတစ်ခါမှာတော့ကျွန်တော်တို့ echo system တွေအကြောင်းကိုပြောပြပေးသွားမှာပဲဖြစ်ပါတယ်။နောက်ထပ် node.js တွေက JavaScript တွေကောင်းမွန်တဲ့နေရာရောက်ဖို့အတွက်ဘယ်လိုကူညီလဲဆိုတာတွေကိုပါလေ့လာရမှဖြစ်ပါတယ်။ Discover applications တွေ tools တွေကို JavaScript တွေနဲ့တည်ဆောက်ထားပါတယ်။ကျွန်တော်တို့တွေက popular ဖြစ်တဲ့ javaScript frameworks တွေ libraries တွေကိုပိုပြီးသိထားဖို့လိုပါတယ်။JavaScript ရဲ့မူလ design ကတော့ browser ထဲမှာ run တာမျိုးပဲဖြစ်ပါတယ်။ဒါပေမယ့် အခြားနေရာတွေမှာလည်းပဲကျွန်တော်တို့ရှာတွေနိုင်ပါတယ်။ ဥပမာ JavaScript က web-based programming language တစ်ခုသာဖြစ်ပြီးအဲ့တာက front end ကော back end နှစ်ခုစလုံးပေါ်မှာ run နိုင်တာပဲဖြစ်ပါတယ်။အဲ့တာကဘာကိုဆိုလိုသလဲဆိုတော့ javaScript ကိုရေးပြီး front end မှာ run တာ သို့မဟုတ် client side ဆိုတာကတော့ ကျွန်တော်တို့က JavaScript တွေကိုရေးပြီး browser ထဲမှာ run တာကိုဆိုလိုပါတယ်။Client-side javaScript တွေက web pages ပေါ်မှာရှိတဲ့ user တွေရဲံ interactivity ပေါ်မှာ allow ပြုလုပ်ပေးပါတယ်။client-side JavaScript တွေမှာအဓိကအားဖြင့် rendered web page တွေရဲ့ အပြုအမူတွေ အသွင်အပြင်တွေကို တိုးတက်စေဖို့ တိုးမြှင့်စေဖို့အတွက်ပါဝင်တာပဲဖြစ်ပါတယ်။back end သို့မဟုတ် server-side ကိုရေးတယ်ဆိုတာကတော့ code တွေကိုရေးနေမဲ့အစား browser ထဲမှာ executer လုက်တာမျိုးပဲဖြစ်ပါတယ်။ sever side javaScript တွေကတော့ APIs တွေ, Database တွေ, web server တွေအတွက်ရေးပေးရတာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့ site ရဲ့ rendered page တစ်ခုတည်းကိုပဲအလုပ်လုပ်နေမယ့်အစား server-side JavaScript ကိုအသုံးပြုပြီး websites ကနေ user တွေရဲ့ request တွေတစ်ခုဆီအတွက်သူတို့ကြိုက်နှစ်သတ်နိုင်မယ့်ပုံစံအတိုင်းဖော်ပြပေးနိုင်ပါတယ်။Application တစ်ခုကို JavaScript တွေနဲအသုံးပြုပြီး front end တွေ back end တွေနှစ်ခုလုံးနဲ့ရေးတာမျိုးကို Full-Stack development လို့ခေါ်ပါတယ်။ host object တွေက environment အားဖြင့် javaScript တွေကိုထောက်ပံ့ပေးပါတယ်။

Node.js

client side JavaScript တွေကို browser တွေနဲ့အလုပ်လုပ်ပါတယ်။ server side JavaScript ကတော့ web server တွေနဲ့အလုပ်လုပ်ပါတယ်။ အဲ့တာကြောင့် javaScript တွေက web server တွေပေါ်မှာဘယ်လိုအလုပ်လုပ်သလဲ။အမြဲတမ်းလိုလို Node.js ကိုအသုံးပြုကြပါတယ်။ Node တွေက websites တွေအပြန်အလှန်လုပ်ဆောင်နိုင်တာတွေထက် javaScript တွေကိုအသုံးပြုပြီး developer တွေကိုခွင့်ပြုပေးပါတယ်။Node တွေက javaScript တွေနဲ့ရေးထားတဲ့ server-side applications တွေဖွံ့ဖြိုးဖို့အတွက် cross platform runtime environment တစ်ခုပဲဖြစ်ပါတယ်။ Node တွေက browser ရဲ့အပြင်ဘက်မှာ javaScript ကိုအလုပ်လုပ်နိုင်တဲ့နည်းလမ်းတစ်ခုဖြစ်အောင်ထောက်ပံ့ပေးနိုင်သောကြောင့်ကျွန်တော်တို့တွေက commandline tools တွေကနေ frameworks တွေ web servers တွေတည်ဆောက်နိုင်ပါတယ်။ Node.js တွေက Google Chrome's V8 ရဲ့ open source JavaScript engine ပေါ်မှာအခြေခံတာပဲဖြစ်ပါတယ်။ Chrome browser နဲ့ Node တွေကတူညီတဲ့ JavaScript engine ကိုအသုံးပြုကြပါတယ်။ဒါပေမယ့်သူတို့မှာမတူညီတဲ့ run time environments တွေပဲဖြစ်ပါတယ်။နောက်ထပ် node တွေက browser တွေကို host objects တွေနဲ့နှိုင်းယှဉ်ပြီး javaScript တွေကိုထောက်ပံ့ပေးပါတယ်။real-time multi-user application တွေကို node နဲ့အမှန်တကယ်တည်ဆောက်ပြီးအလုပ်လုပ်နိုင်ပါတယ်။နောက်ပြီးအဲ့တာတွေက event တွေအများကြီးကို သို့မဟုတ် user actions တွေကိုတစ်ချိန်တည်းမှာအလုပ်လုပ်နိုင်ဖို့အတွက် handle ပြုလုပ်ပေးပါတယ်။multiplayer games, collaborations tools တွေ video-conferencing apps တွေစတာမျိုးတွေပဲဖြစ်ပါတယ်။Node ရဲ့အကြောင်းထဲကအကောင်းမွန်ဆုံးအပိုင်းတစ်ခုကတော့ free-open source tools တစ်ခုရဲ့ driving ecosystem ပဲဖြစ်ပါတယ်။Npm ,node
