Why use Node.js

ကျွန်တော်တို့ javaScript တွေကိုဘာ့ကြောင့်အသုံးပြတာလဲဆိုတာကိုလေ့လာသွားမှာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့က javaScript တွေကို server side ပေါ်မှာရေးနိုင်ပါတယ်။ူဥပမာကျွန်တော်တို့မှာ assistant တစ်ယောက်ရှိမယ်ဆိုရင် invoice တွေရဲ့ stack တွေကိုကျွန်တေ်ာတို့ကပေးရပါမယ်။အဲ့ဒီအခါ့မှာ assistant ကပထမဆုံး invoice ကိုပြန်ပေးပို့လာပြီးတော့ သူရဲ့ payment အတွက်စောင့်နေပါတယ်။ပထမဆုံး payment ကိုရရှိတာနဲ့ bank ထဲမှာသွားစုထားပါတယ်။အဲ့ဒီနောက်မှာမှ နောက်ထပ် Invoice တစ်ခုကိုထပ်ပို့ပါတယ်။အဲ့တာတွေကကျွန်တော်တို့အတွက်တော့မကောင်းပါဘူး။ကျွန်တော်တို့ရဲ့ payment ပေါ်မှာပဲလက်ခံပြီးကျွန်တော်တို့ကိုအလေးမထားရာရောက်ပါတယ်။အဲ့တာကြောင့်ကျွန်တော်တို့ကနောက်တစ်ယောက်ထပ်လဲပြီးခန့်ထားတဲ့အခါမှာတော့ သူက invoice တွေအကုန်လုံးကိုပို့ပေးတာပဲဖြစ်ပါတယ်။payment လာတဲ့အချိန်မှလက်ခံပြီးယူတာပဲဖြစ်ပါတယ်။ဒါပေမယ့်ကျွန်တော်တို့ရဲ့ customer တွေကအခြားသူတွေထက်မြန်မြန်လိုချင်ကြပါတယ်။အဲ့တာကြောင့်ကျွန်တေ်ာတို့ကစောင့်မနေပဲ order တွေကိုအတူတူတစ်ခါတည်းပဲပေးပို့ရမှာဖြစ်ပါတယ်။အဲ့တာွကြောင့်ကျွန်တေ်ာတို့က ဒုတိယ assistant နဲ့အဆင်ပြေပြီး customer တွေနဲ့လည်း relationship တွေပိုတိုးလာပါတယ်။language တွေအများစုနဲ့ frameworks တွေက first assistant လိုပဲအလုပ်လုပ်ပါတယ်။process တွေနောက်ထပ်တစ်ခောက်ထပ်ပြီးမလုပ်ခင်အထိအစကနေအဆုံးအထိ tasks တစ်ခုတည်းကိုပဲ handling ပြုလုပ်ပါတယ်။language တွေမှာဆိုရင် request ပြုလုပ်တဲ့အရာတွေကိုအစဉ်လိုက်ပဲပြုလုပ်ပေးပါတယ်။အဲ့တာတွေရဲ့ ဥပမာကတော့ worker လို့ပဲခေါ်ပါတယ်။Node.js ကတော့ second assistant လိုအလုပ်လုပ်ပါတယ်။တစ်ခါတစ်ရံမှာ Node.js worker တစ်ခုကအခြား language တွေထက်ပိုပြီး responsive ဖြစ်အောင်ပြုလုပ်နိုင်ပါတယ်။Node.js က non-blocking ဖြစ်ပြီးတော့အခြား language တွေ platform တွေပေါ်မှာထက် node.js platform ပေါ်မှာပဲ javaScript application တွေကိုတည်ဆောက်ကြပါတယ်။ဘာ့ကြောင့်လဲဆိုတော့အဲ့တာကပိုမြန်ပြီး efficient ပိုဖြစ်ပါတယ်။

Making a GET Request with https

ကျွန်တော်တို့ programming တွေကိုစပြီးလေ့လာသွားတော့မှာဖြစ်ပါတယ်။ကျွန်တော်တို့ဥပမာထဲမှာ ပထမဆုံးစပြီးရေးပြထားတဲ့ print message ဆိုတဲ့function ကတော့ console ထဲမှာ message တွေကိုထုပ်ပြပေးမှာပဲဖြစ်ပါတယ်။အဲ့ဒီထဲမှာပါတဲ့ argument တွေကတော့ username ,badgeCount, point တွေပဲဖြစ်ပါတယ်။အဲ့တာကို constant message ထဲမှာထည့်ပြထားတာပဲဖြစ်ပါတယ်။ အဲ့ဒီနောက်မှာတော့ console.log နဲ့ထုပ်ပေးမှာပဲဖြစ်ပါတယ်။နောက်ထပ်တစ်ခုကတော့ URL တွေနဲ့ချိတ်ဆက်ပေးရမှာဖြစ်ပါတယ်။အဲ့တာကြောင့်ကျွန်တော်တို့က node.js page ထဲက documentation ထဲကိုသွားပြီးတော့ https ရဲ့ documentation ကိုကြည့်ရမှာပဲဖြစ်ပါတယ်။ဘာ့ကြောင့် http မဟုတ်ပဲ https ကိုအသုံးပြုလည်းဆိုတော့ကျွန်တေ်ာတို့က secure ဖြစ်တဲ့ http site ကိုလက်ခံပေးမှာမို့ပဲဖြစ်ပါတယ်။unsecure URL တွေနဲ့ချိတ်ဆက်မယ်ဆိုရင်တော့ https မဟုတ်ပဲ http ကိုသာအသုံးပြုရမှာပဲဖြစ်ပါတယ်။documentation ထဲမှာတော့ method တွေကိုတွေ့ရမှာဖြစ်ပြီးကျွန်တော်တို့က https get method ကိုအသုံးပြုမှာဖြစ်လို့ get method ထဲကိုဝင်ကြည့်ရပါမယ်။အဲ့ဒီထဲမှာတော့ကျွန်တော်တို့ connect ပြုလုပ်ချင်တဲ့ url တွေကို string တွေအဖြစ်ထားပေးရမှာပဲဖြစ်ပါတယ်။နောက်ပြီး response တွေကိုလည်း return ပြန်ပေးပါတယ်။ console.dir ကိုအသုံးပြုတာကတော့ bit overkill ဖြစ်တဲ့အပေါ်မှာ response object တွေကို log out ပြုလုပ်ပေးဖို့ဖြစ်ပါတယ်။documentation တွေအဆင်သင့်မဖြစ်သေးခင်မှာအဲ့တာတွေကဖြစ်နိုင်ပြီးတော့ properties တွေကိုမြင်နိုင်ဖို့အတွက်နဲ့ debugging ပြုလုပ်နိုင်ဖို့အတွက်ကူညီပေးပါတယ်။ကျွန်တော်တို့ဥပမာထဲမှာတော့ teamtreehouse ထဲမှာရှိတဲ့ dir တွေကိုထုပ်ပြထားတာဖြစ်ပါတယ်။နောက်ထပ်တစ်ခုကတော့ status code တွေဘယ်လောက်ရှိသလဲဆိုတာကိုထုပ်ပြထားတာပဲဖြစ်ပါတယ်။ အဲ့ဒီ file ကိုကူးယူပြီး terminal ထဲကနေလည်းကျွန်တော်တို့ run ကြည့်နိုင်ပါတယ်။
