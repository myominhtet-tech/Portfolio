What is a Common Table Expression

database ထဲမှာ information တွေကိုသိုလှောင်ဖို့အတွက် SQL တွေရေးခြင်းကမကြာခနလိုအပ်နေတာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့ဒီတစ်ခါမှာတော့ queries တွေကိုလွယ်အောင်ဘယ်လိုရေးပြီးတော့ ဘယ်လိုလွယ်ကူအောင်တည်ဆောက်မယ်ဆိုတဲ့အကြောင်းတွေကိုလေ့လာသွားမှာပဲဖြစ်ပါတယ်။အဲ့တာကို Common Table Expression သို့ CTE လို့ခေါ်ပါတယ်။CTE တွေကလေ့လာဖို့အတွက်အမှန်တကယ်ကိုရိုးရှင်းတာပဲဖြစ်ပါတယ်။popular ဖြစ်တဲ့ database တွေကိုထောက်ပံ့နိုင်ပြီးတော့ Oracle, SQL Server တွေနဲ့ PostgreSQL ,Redshift, SQLite, တွေနဲ့ MySQL version 8 နဲ့အထက်တွေစတာတွေမှာထောက်ပံ့နိုင်မှာပဲဖြစ်ပါတယ်။CTE တွေက query တွေထက်ာဘာမှပိုတာမဟုတ်ပါဘူး။အဲ့တာတွေကိုကျွန်တေ်ာတို့ရဲ့နည်မည်တွေကို longer query တွေထဲမှာပြန်ပြီးအသုံးပြုနိုင်အောင်ပြုလုပ်တာပဲဖြစ်ပါတယ်။ကျွန်တော်တို့က CTE ကိုအသုံးပြုဖို့အတွက်ပထမဆုံး WITH ဆိုတဲ့ keyword ကိုသုံးပြီးတော့နောက်မှာ name လိုက်ပေးရတာပဲဖြစ်ပါတယ်။အဲ့ဒီနောက်မှာ AS ဆိုတဲ့ keyword လိုက်ပြီးတော့ parenthesis ထဲမှာတော့ SELECT statement တွေကိုရေးပေးရမှာပဲဖြစ်ပါတယ်။query တွေရဲ့ result return တွေအတွက်တော့ CTE name တွေက alias တွေနဲ့ပုံစံတူပါတယ်။အဲ့ဒီနောက်ကျွန်တော်တို့က CTE name ကို table name လိုပဲအသုံးပြုနိုင်ပါတယ်။CTE တွေက table နဲ့ပုံစံတူပြီးတော့ short query တွေရေးဖို့အတွက်တော့ကျွန်တော်တို့အသုံးမပြုသင့်ပါဘူး။နောက်ပြီး CTE မှာ organizing တွေနဲ့ complex SQL queries တွေအတွက်အကျိုးသတ်ရောက်မှုရှိတဲ့ several key တွေကိုထောက်ပံ့ပေးပါတယ်။ 


    --create CTEs using the WITH satement
    With cte_name AS {
        --select query goes here
    )

    --use CTEs like a table
    SELECT * FROM cte_name

Using Multiple CTEs in a Query

နောက်ထပ်ကျွန်တော်တို့က CTE တွေအများကြီးအသုံးပြုနိုင်ပါတယ်။common table expression တွေကို subqueries တွေကို refactor ပြုလုပ်ဖို့အတွက်နဲ့ readable SQL တွေကိုပိုပြီးတည်ဆောက်ဖို့အတွက်ပိုပြီးအသုံးပြုတာပဲဖြစ်ပါတယ်။subquery ကို inner query လို့လည်းခေါ်ပြီးတော့ nested query လို့လည်းခေါ်ပါတယ်။နောက်ထပ် SELECT keyword ကိုအသုံးပြုပြီးတော့ result set တွေကို generate ပြုလုပ်ပေးနိုင်ပါတယ်။subqueries တွေကို where clause ထဲမှာအခြား query ထဲကနေ information တွေကိုအခြေခံပြီးတော့ result ကိုစစ်ထုပ်ဖို့အတွက် အသုံးပြုနိုင်ပါတယ်။ကျွန်တော်တို့က CTE တစ်ခုထက်ပိုပြီးတော့တည်ဆောက်နိုင်ပါတယ်။အဲ့လိုတည်ဆောက်တဲ့အခါမှာတော့ပထမ CTE ပြီးတဲ့အခါမှာ comma ခြားပြီးမှတည်ဆောက်ရမှာပဲဖြစ်ပါတယ်။နောက်ထပ် CTE မှာတော့ with ဆိုတဲ့ keyword ကိုအသုံးပြုစရာမလိုတော့ပါဘူး။

Convert a Subquery to a CTE

ကျွန်တော်တို့ common table expression တွေက queries တွေကိုရေးတဲ့အခါမှာဘယ်လိုတွေကူညီပြီးတော့လွယ်ကူစေတယ်ဆိုတာကိုကျွန်တော်တို့လေ့လာခဲ့ပြီးဖြစ်ပါတယ်။CTE တွေကကျွန်တော်တို့ရဲ့ queries တွေကို organize ပြုလုပ်ဖို့အတွက်လည်းပဲကူညီပါတယ်။နောက်ပြီး single result set တွေကနေ data တွေကို analyze ပြုလုပ်ဖို့အတွက်လိုအပ်ရင်လည်းပဲအမှန်တကယ်ကိုကူညီပေးပါတယ်။Data တွေက tables တွေအများကြီးထဲမှာပါဝင်ပြီးတော့ database ထဲမှာ store ပြုလုပ်ပေးတာပဲဖြစ်ပါတယ်။
